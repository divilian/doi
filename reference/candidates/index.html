    smeli.candidates¶

    Local helpers for canonicalizing, scoring, comparing, and merging Smeli candidate records.

    Candidate conversion, scoring, and merging helpers.

    A candidate is Smeli's normal paper-record dictionary. It may represent a journal article, conference paper, arXiv preprint, dataset, software record, or another scholarly output. Common keys include title, authors, year, venue, publisher, doi, arxiv_id, openalex_id, url, cited_by_count, metadata_sources, score, and citation_score.

    The public functions in this module are pure local helpers: they do not perform network requests.

    candidate_score ¶

    candidate_score(candidate: dict[str, Any], *, author: str | None = None, title: str | None = None, year: str | int | None = None) -> tuple[int, int]

    Return a local relevance score for a paper candidate.

    Parameters:

    Name Type Description Default
    candidate dict[str, Any]

    A Smeli candidate dictionary.

    		 required
    author str | None

    Optional author name or fragment supplied by the user.

    		 None
    title str | None

    Optional title or title fragment supplied by the user.

    		 None
    year str | int | None

    Optional publication year supplied by the user.

    		 None

    Returns:

    Type Description
    tuple[int, int]

    tuple[int, int]: A (score, citation_bonus) tuple. score reflects local bibliographic relevance; citation_bonus is a weak tie-breaker derived from citation count.
    Notes

    The score is intentionally heuristic. It is useful for ranking plausible candidates, not for measuring scholarly relevance in an academic sense.

    merge_candidates(primary: dict[str, Any], secondary: dict[str, Any]) -> dict[str, Any]

    Merge two candidate dictionaries describing the same paper.

    Parameters:

    Name Type Description Default
    primary dict[str, Any]

    The preferred candidate record. Existing values are generally kept.

    		 required
    secondary dict[str, Any]

    A second candidate record used to fill gaps and add sources.

    		 required

    Returns:

    Type Description
    dict[str, Any]

    dict[str, Any]: A canonicalized candidate dictionary containing the best available fields from both inputs.
    Notes

    This function does not first verify that the records match. Call candidates_are_same() before merging untrusted pairs.
