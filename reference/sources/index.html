    smeli.sources¶

    Metadata source lookups and cross-source orchestration.

    This module is the main networked public API for Smeli. It queries OpenAlex, Crossref, DataCite, arXiv, and doi.org, then returns either structured metadata dictionaries, ORCID lists, BibTeX strings, or Smeli candidate dictionaries.

    Functions return None or an empty list when no useful result is found. Expected misses are handled quietly where practical; real network and parsing errors may print short diagnostic messages.

    get_paper_candidates ¶

    get_paper_candidates(author: str | None = None, title: str | None = None, year: str | int | None = None, identifier: str | None = None, query: str | None = None, *, progress: Callable[[str], None] | None = None) -> list[dict[str, Any]]

    Search all configured metadata sources and return merged candidates.

    Parameters:

    Name Type Description Default
    author str | None

    Optional author name or fragment for fielded search.

    		 None
    title str | None

    Optional title or title fragment for fielded search.

    		 None
    year str | int | None

    Optional publication year for filtering/scoring.

    		 None
    identifier str | None

    Optional DOI, arXiv ID/URL, ORCID, or OpenAlex Work ID/URL. When supplied, identifier resolution takes precedence over fielded search.

    		 None
    query str | None

    Optional free-form bibliographic query. Used when no identifier is supplied.

    		 None
    progress Callable[[str], None] | None

    Optional callback for source-level progress messages. The CLI passes print here; ordinary Python API calls are quiet by default.

    		 None

    Returns:

    Type Description
    list[dict[str, Any]]

    list[dict[str, Any]]: A score-sorted list of merged Smeli candidate dictionaries. Duplicate paper records from multiple sources are collapsed and annotated with metadata_sources.
    Notes

    This is the highest-level public lookup API. It does not print during ordinary Python API use unless a progress callback is supplied.

    Parameters:

    Name Type Description Default
    identifier str | None

    A DOI, DOI URL, arXiv ID/URL, or OpenAlex Work ID/URL.

    		 required

    Returns:

    Type Description
    dict[str, Any] | None

    dict[str, Any] | None: A source-agnostic metadata dictionary when Smeli finds usable metadata, or None when no configured source has a usable record.
    Notes

    DOI lookup tries Crossref first, then DataCite, then OpenAlex. arXiv and OpenAlex identifiers are resolved through their native APIs. Source- specific helpers remain available when callers deliberately want a particular service.

    Parameters:

    Name Type Description Default
    value Any | None

    A DOI, ORCID iD/URL, or a nested metadata/candidate record that may contain ORCID values.

    		 required

    Returns:

    Type Description
    list[str]

    list[str]: Normalized bare ORCID iDs, deduplicated in discovery order.
    Notes

    DOI lookup checks OpenAlex, Crossref, and DataCite. Source-specific helpers return richer {"name": ..., "orcid": ...} records; this high-level helper returns just the normalized identifiers.

    search_orcids ¶

    search_orcids(name: str | None, affiliation: str | None = None, max_results: int = 10) -> list[dict[str, Any]]

    Search for author ORCID iDs by partial author name.

    Parameters:

    Name Type Description Default
    name str | None

    Author name or name fragment to search for.

    		 required
    affiliation str | None

    Optional institution or affiliation fragment used for local filtering of returned author candidates.

    		 None
    max_results int

    Maximum number of ORCID-bearing candidate records to return.

    		 10

    Returns:

    Type Description
    list[dict[str, Any]]

    list[dict[str, Any]]: Candidate author records with keys such as name, orcid, orcid_url, openalex_id, affiliation, affiliations, works_count, cited_by_count, source, and sometimes relevance.
    Notes

    This searches OpenAlex author records. It returns candidate people, not papers, and deliberately excludes author records without ORCID iDs.

    get_paper_candidates_from_orcid ¶

    get_paper_candidates_from_orcid(orcid: str) -> list[dict[str, Any]]

    Fetch paper candidates for an author ORCID using OpenAlex.

    Parameters:

    Name Type Description Default
    orcid str

    A bare ORCID iD or ORCID URL.

    		 required

    Returns:

    Type Description
    list[dict[str, Any]]

    list[dict[str, Any]]: A list of Smeli candidate dictionaries, sorted by OpenAlex publication date descending before later Smeli ranking/merging.
    Notes

    ORCID identifies a person, not a paper, so this function returns a list of papers. OpenAlex may return no papers for a valid ORCID when its author links lack that ORCID.
